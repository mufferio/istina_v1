# Contributing Guide

First off, thank you for considering contributing to this project 🎉  
We welcome contributions of all kinds, including bug fixes, new features, documentation improvements, and refactoring.

---

## 📌 Getting Started

1. Fork the repository (If you are not added as a collaborator)
2. Clone your fork locally
3. Create a new branch (see **Branch Naming Conventions** below)
4. Make your changes
5. Commit your changes (see **Commit Message Format** below)
6. Push to your fork
7. Open a Pull Request

---

## 🌿 Branch Naming Conventions

Please name your branches as:
- `<label>/<milestone>/issue-<number>`
### Examples:
- `infra/m1/issue-1`
- `feat/m3/issue-67`
- `bug/m9/issue-325`

---

## 📝 Commit Message Format

Please follow a consistent commit message format:
- `<type>: <short description> (#<issue>)`
### Examples:
- feat: add user authentication (#41)
- refactor: GeminiProvider migration (#9)

---
## 🔍 Pull Request Guidelines

Before opening a PR, please ensure:

- Your branch is up to date with the main branch
- Your code compiles and runs correctly
- You have tested your changes
- You have followed the coding style of the project
- Your commit messages follow the required format

---

## ✅ Pull Request Checklist

Please complete the following before submitting your PR:

- [ ] I have read the contributing guidelines
- [ ] My code follows the project’s style guidelines
- [ ] I have tested my changes
- [ ] I have updated documentation (if applicable)
- [ ] My commits follow the required format
